# 밥친구 — 기획 문서

> **프로젝트 유형:** GitHub Pages 정적 웹앱 (모바일 퍼스트)
> **기술 스택:** HTML / CSS / JavaScript (프레임워크 자유, 권장: Vanilla 또는 React)
> **서버/API:** 없음 — 지도·채팅·결제 등은 목업(mock) UI로 구현
> **배포:** GitHub Pages (`https://<username>.github.io/<repo>`)
> **대응 우선순위:** 1차 모바일 → 이후 필요시 PC 확장

---

## 1. 프로젝트 개요

"밥친구"는 **혼밥 대신 같이 먹을 친구를 찾아주는 소셜 다이닝 앱**이다. AI 메뉴 추천, 식당 랭킹, N빵 결제, 지역 커뮤니티, 방문도장 리워드 등을 통해 식사를 중심으로 한 사회적 연결을 만든다.

본 프로젝트는 **외주 포트폴리오/시연용**으로, 실제 백엔드 없이 프론트엔드 UI와 인터랙션만 구현한다.

## 2. 디자인 시스템

### 2.1 색상 팔레트

| 토큰 | HEX | 용도 |
|------|------|------|
| `--color-cream` | `#f2efdf` | 배경, 카드 배경 |
| `--color-orange` | `#f29f05` | 주요 액센트(CTA 버튼, 뱃지, 하이라이트) |
| `--color-red` | `#a31f11` | 보조 액센트(경고, 강조 텍스트, VIP 뱃지) |
| `--color-black` | `#1a1a1a` | 텍스트, 아이콘 |
| `--color-gray-600` | `#666666` | 보조 텍스트 |
| `--color-gray-300` | `#d1d1d1` | 구분선, 비활성 요소 |
| `--color-white` | `#ffffff` | 카드, 입력 필드 배경 |

### 2.2 타이포그래피

- 본문: Pretendard 또는 Noto Sans KR (400/500/700)
- 제목/강조: 동일 폰트, Bold(700) + 크기 차이로 위계 표현
- 기본 사이즈: 14px (본문), 18px (섹션 제목), 24px (페이지 제목)

### 2.3 컴포넌트 스타일

- **모서리:** 12px radius (카드), 24px radius (버튼, 칩), 50% (아바타)
- **그림자:** `0 2px 8px rgba(0,0,0,0.08)` (카드 기본)
- **간격 체계:** 4px 배수 (8, 12, 16, 20, 24, 32)
- **바텀 네비게이션:** 고정, 높이 64px, 아이콘 + 라벨, 5탭

## 3. 정보 구조 (IA) 및 네비게이션

메인 화면은 하단 탭 바 5개 섹션으로 구성된다.

```
메인화면
├── 홈 (기본 랜딩)
├── 지도
├── 지역게시판
├── 채팅
└── 마이페이지
```

### 3.1 탭 별 하위 기능 트리

**홈**
- 검색 기능 (식당, 친구, 지역 통합 검색)
- 식당 랭킹
- 알림
- AI 메뉴 추천 기능
- 추천 식당 리스트
- 추천 친구 리스트
- 방문 도장 (리워드)
- 이벤트 배너

**지도**
- N빵 결제 (더치페이 모의 화면)
- 같이 주문
- 친구 매칭
- 시계 (약속 타이머)

**지역게시판**
- 지역 맛집 게시판
- 지역 이벤트
- 공지사항 (앱 업데이트)

**채팅**
- 알림
- 채팅창 (1:1 및 그룹)

**마이페이지**
- 최근에 식사한 친구
- 즐겨찾기 친구
- 설정
- 개인정보 관리
- 식당 랭킹
- 1대1 문의
- 도움말
- 알림 설정
- 언어 설정

## 4. 페이지별 상세 기획

### 4.1 홈 화면

**레이아웃** (위→아래 스크롤)

1. **헤더 영역**
   - 인사말: "안녕하세요" + 줄바꿈 + "오늘은 뭐 드실래요?" (Bold, 24px)
   - 통합 검색 바: 마이크 아이콘(좌), placeholder "식당, 친구, 지역 검색", 필터 아이콘(우)

2. **AI 메뉴 추천 카드**
   - 배경: `--color-cream` 또는 그라디언트
   - 타이틀: "오늘 뭐 먹지?" → "AI 메뉴 추천" (Bold, 큰 사이즈)
   - 설명: "지금 기분과 위치 기반으로 딱 맞는 메뉴를 골라드려요"
   - CTA 버튼: "추천받기 →" (검정 배경, 흰 텍스트, pill shape)
   - **동작(mock):** 탭 시 로딩 애니메이션 → 더미 추천 결과 3~4개 표시

3. **추천 식당 섹션**
   - 섹션 헤더: "추천 식당" + 부제 "밥친구님을 위한 오늘의 픽" + "더 보기→"
   - 가로 스크롤 카드 리스트 (3장 이상)
   - 카드 구성: 식당 이미지(placeholder gray), 별점(★ 4.8), 북마크 아이콘, 카테고리 태그("한식, 곱창"), 식당명, 거리(0.3 Km), 가격대(₩)

4. **방문 도장 카드**
   - 진행도: "7/10" (큰 Bold 숫자)
   - VIP 뱃지 (우상단, `--color-orange` 배경)
   - 설명: "다음 보상 · 무료 음료 쿠폰"
   - 도장 그리드: 10칸, 7개 체크 완료(✓), 3개 빈칸

5. **추천 친구 섹션**
   - 섹션 헤더: "추천 친구" + 부제 "비슷한 취향을 가진 사람들" + "더 보기→"
   - 가로 스크롤 카드 리스트
   - 카드 구성: 프로필 이미지(원형), 취향 태그("라멘, 일식"), 이름, "친구 추가" 버튼(`--color-cream` 배경)

6. **이벤트 배너**
   - 풀 와이드 카드, 어두운 배경
   - 라벨: "이번 주 이벤트"
   - 타이틀: 예시 "성수동 푸드트럭 페스티벌" (Bold, 큰 사이즈)
   - 설명: "주말 양일간 30개 푸드트럭이 모입니다"
   - CTA: "참여하기 →" (pill shape 버튼)

**인터랙션 메모:**
- 검색 바 탭 → 검색 화면(별도 페이지 또는 오버레이)으로 이동
- 카드 탭 → 식당 상세 페이지(mock)
- "더 보기" → 리스트 전체 페이지
- 모든 데이터는 하드코딩된 JSON 또는 JS 객체에서 로드

### 4.2 지도 화면

**레이아웃**

1. **상단 바**
   - 위치 선택기: 📍 아이콘 + "논현동" + 드롭다운 화살표
   - 카테고리 필터 칩: "주변"(활성, 검정 배경), "한식", "양식", "일식" 등 (가로 스크롤)

2. **지도 영역**
   - 전체 화면 너비, 상단 절반 이상 차지
   - **구현 방식(mock):** 정적 이미지 또는 CSS로 만든 회색 placeholder 영역. 핀 마커는 CSS로 위에 올림
   - 실제 지도 API(Kakao/Naver/Google Maps) 없이 시각적 표현만

3. **바텀 시트 (Draggable Sheet)**
   - 상단 핸들 바 (드래그 인디케이터)
   - 탭 메뉴: "식당 랭킹" | "같이 주문" | "N 결제" | ... (가로 스크롤)
   - 선택된 탭 내용이 아래에 표시

4. **식당 랭킹 탭 (기본)**
   - 식당 리스트 카드 (세로 스크롤)
   - 카드 구성: 식당 썸네일(정사각형, 좌측), 별점(★ 4.8), 식당명, 거리·가격, 북마크 아이콘(우측)
   - 하단 우측: FAB(＋) 버튼 — 식당 등록 또는 리뷰 작성 진입

5. **같이 주문 탭**
   - 현재 모집 중인 같이 주문 리스트
   - 카드: 모집자 프로필, 식당명, 현재 인원/최대 인원, 마감 시간, "참여하기" 버튼

6. **N 결제 탭**
   - 최근 N빵 기록 또는 새 N빵 만들기 버튼
   - 금액 입력, 인원수 입력 → 1인당 금액 자동 계산 (JS로 구현 가능)

**인터랙션 메모:**
- 바텀 시트는 위/아래 드래그로 높이 조절 (CSS + JS)
- 카테고리 칩 탭 시 활성 상태 토글 (필터링은 시각적 전환만)
- 위치 드롭다운 → 동네 선택 모달 (하드코딩 목록)

## 5. Mock 데이터 구조

서버 없이 운영하므로, 모든 데이터는 JS 파일 또는 JSON으로 하드코딩한다.

```
/data
├── restaurants.json      # 식당 목록 (이름, 카테고리, 별점, 거리, 이미지 placeholder)
├── users.json            # 추천 친구 목록 (이름, 취향 태그, 프로필 이미지)
├── events.json           # 이벤트 목록
├── posts.json            # 지역게시판 게시글
├── chatRooms.json        # 채팅방 목록
└── messages.json         # 채팅 메시지 내역
```

```json

  "image": "placeholder",
  "bookmarked": false,
  "description": "40년 전통 노포 곱창집",
  "address": "서울 중구 을지로 xxx"
```

## 6. 라우팅 및 페이지 구조

GitHub Pages는 SPA 라우팅에 제한이 있으므로, 다음 두 가지 방식 중 택 1:

**방식 A — 단일 HTML + JS 뷰 전환 (권장)**
- `index.html` 하나에 모든 뷰를 `<section>` 또는 `<div>`로 포함
- JS로 탭/페이지 전환 시 `display: none/block` 또는 class 토글
- URL 해시(`#home`, `#map`, `#board`, `#chat`, `#mypage`)로 상태 관리

**방식 B — 멀티 HTML**
- `index.html` (홈), `map.html`, `board.html`, `chat.html`, `mypage.html`
- 하단 탭 바에서 페이지 간 이동

### 권장 파일 구조

```
/
├── index.html
├── css/
│   ├── reset.css
│   ├── variables.css       # 색상, 폰트, 간격 토큰
│   ├── components.css      # 공통 컴포넌트 (카드, 버튼, 칩 등)
│   └── pages.css           # 페이지별 레이아웃
├── js/
│   ├── app.js              # 라우팅, 초기화
│   ├── home.js             # 홈 화면 로직
│   ├── map.js              # 지도 화면 로직
│   ├── board.js            # 게시판 로직
│   ├── chat.js             # 채팅 로직
│   └── mypage.js           # 마이페이지 로직
├── data/
│   ├── restaurants.json
│   ├── users.json
│   └── ...
└── assets/
    └── images/             # 아이콘, placeholder 이미지
```

## 8. 인터랙션 및 Mock 동작 명세

### 8.1 실제 동작하는 기능

| 기능 | 구현 방식 |
|------|-----------|
| 탭 네비게이션 | 하단 탭 클릭 시 해당 뷰로 전환 |
| 카테고리 필터 | 칩 클릭 시 활성 토글 + 리스트 필터링 (JSON 데이터 기반) |
| 검색 | 입력값으로 식당/친구 JSON 필터링 후 결과 표시 |
| 북마크 토글 | 아이콘 클릭 시 채움/빈 상태 전환 |
| 방문 도장 | 숫자 변경 시 UI 업데이트 (localStorage 활용) |

## 9. 반응형 및 뷰포트

### 모바일 퍼스트 원칙

> **1차 대응 대상은 모바일이다.** 모든 UI는 모바일 화면에서 가장 깔끔하게 보이도록 설계한다. PC 대응은 이후 필요시 확장하며, 초기 구현에서는 고려하지 않아도 된다.

- **기준 해상도:** 390 × 844px (iPhone 14 기준)
- **지원 범위:** 320px ~ 430px 폭을 기본으로 최적화
- `<meta name="viewport" content="width=device-width, initial-scale=1.0">`
- 터치 대상(버튼, 카드, 탭)은 최소 44×44px 확보
- 텍스트는 모바일에서 읽기 편한 크기 유지 (본문 14px 이상)
- 가로 스크롤 영역은 스냅 포인트 적용, 스크롤바는 숨김 처리
- 하단 safe area 대응: `padding-bottom: env(safe-area-inset-bottom)`

### PC 접속 시 처리 (이후 확장)

- `max-width: 430px` + `margin: 0 auto`로 모바일 프레임 중앙 정렬
- 배경은 `--color-cream` 또는 연한 패턴으로 채움
- 폰 프레임 그림자 등은 선택사항 — 모바일 완성 이후 고려

## 11. GitHub Pages 배포 체크리스트

- `index.html`이 루트에 위치할 것
- 모든 경로는 상대 경로 사용 (`./css/`, `./js/`, `./data/`)
- 이미지는 `/assets/images/`에 정리
- repo Settings → Pages → Source: `main` branch, `/ (root)`
- 404 대응: SPA의 경우 `404.html`을 `index.html` 복사본으로 배치 (해시 라우팅 사용 시 불필요)
