# 밥친구 — STEP 구현 문서

> 기획 문서(`기획문서_밥친구.md`)를 기반으로, 전체 구현을 **10개 STEP**으로 분리한다.
> 각 STEP은 이전 STEP의 완료를 전제로 하며, 독립적으로 동작 확인이 가능한 단위로 구성했다.

### ⚠️ 전체 STEP 공통 원칙: 모바일 퍼스트

> **1차 대응 대상은 모바일이다.** 모든 UI는 모바일(390×844px 기준)에서 가장 깔끔하게 보이도록 설계·구현한다. PC 대응은 이후 필요시 확장하며, 초기 구현 단계에서는 모바일 최적화에만 집중한다.
>
> 이에 따라 모든 STEP에서:
> - 레이아웃과 터치 영역은 모바일 기준으로 설계할 것
> - 반응형 브레이크포인트(PC용 미디어쿼리)는 작성하지 않아도 됨
> - 테스트 시 Chrome DevTools 모바일 뷰(390px 폭)에서 확인할 것

## STEP 1 — 프로젝트 초기 세팅 및 앱 셸

### 구현 목표
프로젝트 폴더 구조를 잡고, 디자인 시스템(CSS 변수)과 글로벌 레이아웃 골격을 완성한다. 이 단계가 끝나면 빈 화면 위에 하단 탭 바가 표시되고, 탭 클릭 시 5개 빈 뷰가 전환되는 상태여야 한다.

### 구현 내용

**1-1. 파일 구조 생성**
```
/
├── index.html
├── css/
│   ├── reset.css            # margin/padding 초기화, box-sizing
│   ├── variables.css        # 색상·폰트·간격·radius 토큰
│   ├── global.css           # body, 앱 컨테이너, 스크롤 설정
│   ├── components.css       # (빈 파일, 이후 STEP에서 채움)
│   └── pages.css            # (빈 파일)
├── js/
│   ├── app.js               # 라우터 초기화, 탭 전환 로직
│   └── router.js            # 해시 기반 라우팅 유틸
├── data/                    # (빈 폴더)
└── assets/images/           # (빈 폴더)
```

**1-2. CSS 변수 정의 (`variables.css`)**
- `--color-cream: #f2efdf`
- `--color-orange: #f29f05`
- `--color-red: #a31f11`
- `--color-black: #1a1a1a`
- `--color-gray-600: #666666`
- `--color-gray-300: #d1d1d1`
- `--color-white: #ffffff`
- `--font-family: 'Pretendard', 'Noto Sans KR', sans-serif`
- `--radius-card: 12px` / `--radius-button: 24px`
- `--shadow-card: 0 2px 8px rgba(0,0,0,0.08)`
- 간격: `--space-xs: 4px` ~ `--space-xl: 32px`

**1-3. 앱 컨테이너**
- `width: 100%`, `min-height: 100vh` — 모바일 전체 화면 기준
- `position: relative`로 바텀 네비게이션 고정 기준점
- PC용 `max-width: 430px` + 중앙 정렬은 이 단계에서 넣지 않음 (이후 PC 확장 시 추가)

**1-4. 하단 네비게이션 바 (`BottomNav`)**
- 고정 하단, 높이 64px, 배경 흰색, 상단 border
- 5개 탭: 홈(🏠), 지도(🗺), 게시판(📋), 채팅(💬), 마이(👤)
- 아이콘은 우선 유니코드 또는 인라인 SVG 사용 (이후 아이콘 라이브러리로 교체 가능)
- 활성 탭: `--color-black` + bold, 비활성: `--color-gray-300`

**1-5. 해시 라우팅**
- URL 해시 변경(`#home`, `#map`, `#board`, `#chat`, `#mypage`)을 감지
- 해당하는 `<section>` 만 `display: block`, 나머지 `display: none`
- 기본값: `#home`

**1-6. 뷰포트 메타 태그**
- `<meta name="viewport" content="width=device-width, initial-scale=1.0">`

### 완료 조건
- [ ] 브라우저에서 `index.html`을 열면 하단 탭 바 5개가 보인다
- [ ] 각 탭 클릭 시 해당 빈 섹션으로 전환되며, URL 해시가 변경된다
- [ ] 뒤로 가기(브라우저 백) 시 이전 탭으로 돌아간다
- [ ] Chrome DevTools 모바일 뷰(390px)에서 레이아웃이 깔끔하게 보인다
- [ ] CSS 변수가 정상 적용된다 (DevTools에서 확인)

## STEP 2 — Mock 데이터 및 공통 컴포넌트

### 구현 목표
전체 앱에서 반복 사용되는 컴포넌트의 CSS 클래스를 만들고, Mock 데이터 JSON 파일을 생성한다. 이 단계가 끝나면 각 컴포넌트를 테스트용으로 화면에 렌더링할 수 있어야 한다.

### 구현 내용

**2-1. Mock 데이터 파일 생성**

`data/restaurants.json` — 식당 8~10개
```json
{
  "id": "r001",
  "name": "을지로 노포 곱창",
  "category": ["한식", "곱창"],
  "rating": 4.8,
  "distance": "0.3 Km",
  "priceRange": "₩",
  "image": "",
  "bookmarked": false,
  "description": "40년 전통 노포 곱창집",
  "address": "서울 중구 을지로 xxx"
}
```

`data/users.json` — 추천 친구 6~8명
```json
{
  "id": "u001",
  "name": "한지호",
  "tags": ["라멘", "일식"],
  "profileImage": "",
  "isFriend": false
}
```

`data/events.json` — 이벤트 3~4개

`data/posts.json` — 게시판 글 5~6개 (작성자, 내용, 좋아요 수, 댓글 수, 시간)

`data/chatRooms.json` — 채팅방 4~5개 (상대 이름, 마지막 메시지, 시간, 안 읽은 수)

`data/messages.json` — 채팅방별 대화 내역 5~10개

**2-2. 공통 컴포넌트 CSS**

각 컴포넌트의 클래스를 `components.css`에 작성:

- `.card` — 기본 카드 (흰 배경, radius, shadow)
- `.card-restaurant` — 식당 카드 (가로형: 썸네일+정보 / 세로형: 이미지 위+정보 아래)
- `.card-friend` — 친구 카드 (원형 이미지, 태그, 이름, 버튼)
- `.card-stamp` — 방문도장 카드 (진행 바, 그리드)
- `.card-event` — 이벤트 배너 (어두운 배경, 타이틀, CTA)
- `.card-chat` — 채팅방 리스트 아이템
- `.card-post` — 게시판 글 카드
- `.chip` / `.chip--active` — 필터 칩
- `.btn` / `.btn--primary` / `.btn--pill` — 버튼 변형
- `.badge` — 알림/VIP 뱃지
- `.search-bar` — 검색 입력 바
- `.section-header` — 섹션 제목 + "더 보기"
- `.fab` — 플로팅 액션 버튼
- `.toast` — 토스트 메시지
- `.modal` / `.modal-overlay` — 모달
- `.menu-item` — 마이페이지 메뉴 항목 (아이콘 + 텍스트 + >)

**2-3. 데이터 로드 유틸**
- `js/data.js` 파일 생성
- `fetch('./data/restaurants.json')` 등으로 JSON 로드하는 비동기 함수
- 로드된 데이터를 전역 상태 객체에 저장

### 완료 조건
- [ ] 모든 JSON 파일이 브라우저에서 fetch로 정상 로드된다
- [ ] 각 컴포넌트 클래스를 테스트 HTML에 적용하면 디자인이 정상 출력된다
- [ ] 카드, 버튼, 칩 등이 기획문서의 디자인 시스템(색상, radius, shadow)을 따른다

## STEP 3 — 홈 화면

### 구현 목표
홈 화면의 전체 레이아웃과 모든 섹션을 구현한다. 스크롤 가능한 피드 형태로, 상단부터 헤더 → 검색바 → AI 추천 카드 → 추천 식당 → 방문도장 → 추천 친구 → 이벤트 배너 순으로 배치한다.

### 구현 내용

**3-1. 헤더 영역**
- 인사말 텍스트: "안녕하세요" (14px, gray) + "오늘은 뭐 드실래요?" (24px, bold, black)
- 좌측 정렬, 상단 패딩 충분히

**3-2. 검색 바**
- 마이크 아이콘(좌) + placeholder 텍스트 + 필터 아이콘(우)
- 클릭 시 검색 오버레이 뷰로 전환 (STEP 8에서 검색 로직 연결, 이 단계에서는 뷰 전환만)

**3-3. AI 메뉴 추천 카드**
- `--color-cream` 배경의 대형 카드
- "오늘 뭐 먹지?" 라벨 + "AI 메뉴 추천" 타이틀 (bold, 큰 사이즈)
- 설명 텍스트 + "추천받기 →" 버튼 (검정 pill)
- 클릭 동작은 STEP 9에서 구현 (이 단계에서는 버튼만 배치)

**3-4. 추천 식당 섹션**
- `.section-header`: "추천 식당" + "밥친구님을 위한 오늘의 픽" + "더 보기→"
- `restaurants.json`에서 상위 3~4개 로드
- 가로 스크롤 컨테이너 (`overflow-x: auto`, `scroll-snap`)
- 세로형 `.card-restaurant` 사용

**3-5. 방문 도장 카드**
- "7/10" 큰 숫자 + "◆ VIP" 뱃지 (우상단, `--color-orange`)
- "다음 보상 · 무료 음료 쿠폰" 설명
- 10칸 그리드: 체크 아이콘 7개 + 빈 원 3개
- 도장 수는 하드코딩 (localStorage 연동은 STEP 9)

**3-6. 추천 친구 섹션**
- `.section-header`: "추천 친구" + "비슷한 취향을 가진 사람들" + "더 보기→"
- `users.json`에서 4명 로드
- 가로 스크롤, `.card-friend` 사용
- "친구 추가" 버튼 포함 (동작은 STEP 9)

**3-7. 이벤트 배너**
- 어두운 배경(#333 또는 그라디언트) 대형 카드
- "이번 주 이벤트" 라벨 + 이벤트 제목 + 설명 + "참여하기 →"
- `events.json`에서 첫 번째 이벤트 로드

**3-8. 바텀 여백**
- 하단 네비게이션에 가려지지 않도록 `padding-bottom: 80px`

### 완료 조건
- [ ] 홈 탭 진입 시 모든 섹션이 세로로 배치되어 스크롤 가능하다
- [ ] 추천 식당, 추천 친구 섹션이 가로 스크롤된다
- [ ] Mock 데이터(식당, 친구, 이벤트)가 카드에 정상 렌더링된다
- [ ] 방문 도장 그리드가 7/10 상태로 표시된다
- [ ] UI 목업(ui_mock.png)의 레이아웃과 시각적으로 유사하다

## STEP 4 — 지도 화면

### 구현 목표
지도 탭의 전체 UI를 구현한다. 상단 위치 선택 + 필터 칩, 가운데 지도 mock 영역, 하단 드래그 가능한 바텀 시트(탭 3개)를 완성한다.

### 구현 내용

**4-1. 상단 바**
- 위치 선택기: 📍 + "논현동" + ▼ 화살표
- 클릭 시 동네 선택 모달 (하드코딩 목록 5~6개: 논현동, 역삼동, 성수동 등)
- 카테고리 칩 가로 스크롤: "주변"(기본 활성), "한식", "양식", "일식", "중식", "카페"
- 칩 클릭 시 활성 토글 (검정 배경 ↔ 흰 배경)

**4-2. 지도 영역 (Mock)**
- 화면 상반부를 차지하는 회색 영역 (`background: #e8e8e8`)
- CSS로 만든 핀 마커 3~4개 (절대 위치)
- 핀 클릭 시 해당 식당 이름 툴팁 표시 (선택)
- 지도 API 없음 — 시각적 placeholder

**4-3. 바텀 시트**
- 드래그 핸들 바 (가운데 회색 막대)
- 터치/마우스 드래그로 시트 높이 조절 (최소·중간·최대 3단계 snap)
- 시트 내부 탭: "식당 랭킹" | "같이 주문" | "N 결제"
- 탭 가로 스크롤

**4-4. 식당 랭킹 탭 (기본 활성)**
- `restaurants.json` 전체 로드, 세로 리스트
- 가로형 `.card-restaurant`: 좌측 썸네일(정사각형) + 우측 정보(이름, 별점, 거리, 가격) + 북마크 아이콘
- 칩 필터 연동: 선택된 카테고리로 리스트 필터링

**4-5. 같이 주문 탭**
- 모집 카드 3~4개 (하드코딩)
- 카드: 모집자 이름, 식당명, "2/4명", 마감 시간, "참여하기" 버튼
- 참여하기 → "참여 완료!" 토스트 메시지 (`.toast`)

**4-6. N 결제 탭**
- 심플 계산기 UI
- 총 금액 입력 (숫자 input) + 인원 수 입력 (숫자 input 또는 +/- 버튼)
- 1인당 금액 실시간 표시 (JS 계산)
- "결제하기" 버튼 → "결제 완료" 토스트

**4-7. FAB 버튼**
- 우하단 고정, 바텀시트 위에 떠 있음
- "+" 아이콘, 클릭 시 모달 (식당 등록 폼 mock)

### 완료 조건
- [ ] 지도 탭 진입 시 상단 필터 + 지도 영역 + 바텀 시트가 보인다
- [ ] 바텀 시트가 드래그로 높이 조절된다 (최소 2단계)
- [ ] 3개 탭(식당 랭킹/같이 주문/N 결제)이 전환된다
- [ ] 카테고리 칩 선택 시 식당 리스트가 필터링된다
- [ ] N빵 계산기에 숫자 입력 시 1인당 금액이 실시간 계산된다
- [ ] UI 목업(ui_mock2.png)의 레이아웃과 시각적으로 유사하다

## STEP 6 — 채팅 화면

### 구현 목표
채팅 리스트 화면과 채팅방 상세 화면을 구현한다. 메시지 입력 시 내 말풍선이 추가되는 기본 인터랙션까지 포함한다.

### 구현 내용

**6-1. 채팅 리스트**
- 상단: "채팅" 타이틀 + 🔔 알림 아이콘
- `chatRooms.json` 로드
- `.card-chat`: 프로필 이미지(원형) + 이름 + 마지막 메시지(1줄 말줄임) + 시간(우상단) + 안 읽은 수 뱃지(우하단, `--color-orange`)
- 그룹 채팅: 프로필 이미지 2~3개 겹침 표현

**6-2. 채팅방 상세 (서브 뷰)**
- 채팅방 클릭 시 리스트 숨기고 채팅방 뷰 표시 (또는 슬라이드 전환)
- 상단 바: ← 뒤로가기 + 상대 이름 + ⋮ 메뉴
- 메시지 영역 (세로 스크롤):
  - 상대 말풍선: 좌측, 흰 배경, 이름+시간 표시
  - 내 말풍선: 우측, `--color-orange` 배경, 흰 텍스트
- `messages.json`에서 해당 채팅방 메시지 로드

**6-3. 메시지 입력**
- 하단 고정 입력 바: 텍스트 input + 전송 버튼(➤)
- 전송 클릭 또는 Enter 시:
  - 내 말풍선 추가 (DOM에 append)
  - 입력 필드 초기화
  - 자동 스크롤 하단으로
- 자동 응답(optional): 1~2초 후 상대 말풍선 하나 추가 ("네 알겠습니다!" 등 랜덤)

**6-4. 뒤로가기**
- 채팅방에서 ← 클릭 시 채팅 리스트로 복귀

### 완료 조건
- [ ] 채팅 탭 진입 시 채팅방 리스트가 표시된다
- [ ] 채팅방 클릭 → 대화 내역이 말풍선으로 렌더링된다
- [ ] 메시지 입력 후 전송 시 내 말풍선이 추가된다
- [ ] 뒤로가기로 리스트 복귀가 가능하다
- [ ] 안 읽은 메시지 뱃지가 표시된다

## STEP 7 — 마이페이지 화면

### 구현 목표
마이페이지의 프로필 헤더와 메뉴 리스트, 그리고 주요 서브 페이지(최근 식사 친구, 즐겨찾기, 설정)를 구현한다.

### 구현 내용

**7-1. 프로필 헤더**
- 프로필 이미지 (큰 원형, 80px, placeholder)
- 닉네임 (bold, 18px) + 한 줄 소개 (gray, 14px)
- "프로필 편집" 버튼 (아웃라인 스타일)

**7-2. 메뉴 리스트**
- `.menu-item` 반복:
  - 최근에 식사한 친구
  - 즐겨찾기 친구
  - 설정
  - 개인정보 관리
  - 식당 랭킹
  - 1대1 문의
  - 도움말
  - 알림 설정
  - 언어
- 각 항목: 아이콘(좌) + 텍스트(중) + ">" 화살표(우)
- 섹션 구분선으로 그룹 구분

**7-3. 서브 페이지 — 최근에 식사한 친구**
- 메뉴 클릭 → 서브 뷰 표시 (← 뒤로가기 + 제목)
- `users.json`에서 3~4명 표시, 각 카드에 "마지막 식사: 2일 전" 등

**7-4. 서브 페이지 — 즐겨찾기 친구**
- 친구 카드 + ★ 즐겨찾기 해제 버튼

**7-5. 서브 페이지 — 설정**
- 토글 스위치 목록: 알림 수신, 위치 서비스, 다크모드
- 토글 클릭 시 on/off 전환 (시각적 전환만, 다크모드는 STEP 9)

**7-6. 서브 페이지 — 1대1 문의**
- 제목 input + 내용 textarea + "문의하기" 버튼
- 버튼 클릭 → "접수되었습니다" 토스트 (실제 전송 없음)

**7-7. 서브 페이지 — 언어**
- 한국어 / English 라디오 버튼
- 선택 시 localStorage에 저장 (실제 번역은 미구현)

### 완료 조건
- [ ] 마이페이지 탭 진입 시 프로필 헤더 + 메뉴 리스트가 표시된다
- [ ] 메뉴 항목 클릭 → 해당 서브 페이지로 전환된다
- [ ] 서브 페이지에서 ← 뒤로가기로 마이페이지 메인 복귀가 가능하다
- [ ] 설정의 토글 스위치가 동작한다
- [ ] 1대1 문의 폼 제출 시 토스트가 표시된다

## STEP 8 — 검색 기능

### 구현 목표
홈 화면의 검색 바를 활성화하고, 식당·친구를 통합 검색하는 오버레이 화면을 구현한다.

### 구현 내용

**8-1. 검색 오버레이 뷰**
- 검색 바 클릭 시 전체 화면 오버레이로 전환
- 상단: ← 뒤로가기 + 검색 input (자동 포커스) + ✕ 클리어 버튼
- 입력 전: "최근 검색어" 리스트 (하드코딩 3~4개) + "인기 검색어" 리스트

**8-2. 실시간 검색 필터링**
- 입력값이 2글자 이상일 때 필터링 시작
- `restaurants.json`의 name, category 필드 대상 검색
- `users.json`의 name, tags 필드 대상 검색
- 결과를 "식당"과 "친구" 섹션으로 분리 표시
- 결과 없을 시: "검색 결과가 없습니다" 안내

**8-3. 검색 결과 카드**
- 식당: 가로형 `.card-restaurant` (간소화 버전)
- 친구: 가로형 `.card-friend` (간소화 버전)
- 카드 클릭 → 해당 상세 뷰(mock, alert 또는 모달)

**8-4. 뒤로가기**
- ← 또는 오버레이 바깥 클릭 → 홈으로 복귀
- 검색 input 초기화

### 완료 조건
- [ ] 홈 검색 바 클릭 시 검색 오버레이가 열린다
- [ ] 텍스트 입력 시 식당·친구 통합 검색 결과가 실시간 표시된다
- [ ] 결과가 없으면 빈 상태 안내가 표시된다
- [ ] 뒤로가기로 홈 복귀가 가능하다

## STEP 9 — Mock 인터랙션 및 부가 기능

### 구현 목표
지금까지 "버튼만 배치"했던 요소들에 인터랙션을 연결하고, 부가 기능(다크모드, AI 추천 mock 등)을 구현한다.

### 구현 내용

**9-1. AI 메뉴 추천 (Mock)**
- 홈의 "추천받기" 버튼 클릭 시:
  - 버튼 → 로딩 스피너 표시 (1.5초)
  - 로딩 완료 → 추천 결과 카드 3~4개 슬라이드업 표시
  - 추천 항목: 하드코딩 메뉴 목록에서 랜덤 선택 (김치찌개, 라멘, 파스타, 치킨 등)
  - "다시 추천받기" 버튼으로 재시도 가능

**9-2. 북마크 토글**
- 식당 카드의 북마크 아이콘 클릭 시 채움/빈 상태 전환
- localStorage에 북마크 ID 배열 저장
- 북마크된 식당은 앱 재로드 시에도 유지

**9-3. 친구 추가 토글**
- "친구 추가" 버튼 클릭 → "추가됨" 텍스트 + 체크 아이콘으로 전환
- 재클릭 시 원래 상태 복귀

**9-4. 방문 도장 localStorage 연동**
- 도장 수를 localStorage에서 읽기/쓰기
- (시연용) 도장 카드 클릭 시 도장 1개 추가, 10개 도달 시 "축하합니다!" 모달

**9-5. 다크모드**
- 마이페이지 설정의 다크모드 토글과 연동
- 토글 시 `<body>`에 `.dark` 클래스 추가/제거
- `.dark` 클래스 하위에서 CSS 변수 오버라이드:
  - `--color-cream → #1a1a1a`
  - `--color-white → #2a2a2a`
  - `--color-black → #f2efdf`
  - 기타 필요한 색상 반전
- 설정 localStorage에 저장, 로드 시 자동 적용

**9-6. 토스트 메시지 시스템**
- 글로벌 `showToast(message)` 함수
- 화면 하단(바텀 네비 위)에 메시지 표시, 2초 후 자동 사라짐
- fade in/out 애니메이션

**9-7. 알림 페이지**
- 홈 또는 채팅의 알림 아이콘 클릭 → 알림 리스트 오버레이
- 하드코딩 알림 5~6개: "김지수님이 친구 요청을 보냈습니다", "을지로 곱창에 새 리뷰가 달렸습니다" 등
- 읽음/안읽음 시각 구분

### 완료 조건
- [ ] AI 추천 버튼 → 로딩 → 결과 표시 플로우가 동작한다
- [ ] 북마크, 친구 추가가 토글되며 localStorage에 저장된다
- [ ] 다크모드 토글 시 전체 앱 색상이 전환된다
- [ ] 토스트 메시지가 정상 표시되고 자동으로 사라진다
- [ ] 알림 리스트가 표시된다

## STEP 10 — 최종 다듬기 및 GitHub Pages 배포

### 구현 목표
전체 앱을 최종 검수하고, 애니메이션·전환 효과를 추가한 뒤 GitHub Pages에 배포한다.

### 구현 내용

**10-1. 전환 애니메이션**
- 탭 전환 시 fade 또는 slide 효과 (`transition: opacity 0.2s`)
- 서브 페이지 진입/복귀 시 슬라이드 좌우 전환
- 카드 클릭 시 press 효과 (`transform: scale(0.98)`)
- 가로 스크롤 영역에 스크롤 스냅 적용

**10-2. 로딩 상태**
- 앱 초기 로드 시 스플래시 화면 (앱 이름 + 로고, 1초)
- 데이터 로드 중 스켈레톤 UI 표시 (선택)

**10-3. 빈 상태 처리**
- 검색 결과 없음, 채팅 없음, 게시글 없음 등에 빈 상태 일러스트 + 안내 텍스트

**10-4. 모바일 우선 점검**
- Chrome DevTools 모바일 뷰(390px)에서 모든 화면 최종 확인
- 실제 모바일 기기(iPhone Safari, Android Chrome)에서 접속 테스트
- iOS Safari 하단 safe area 대응: `padding-bottom: env(safe-area-inset-bottom)`
- 터치 대상이 44×44px 이상인지 재확인
- 가로 스크롤 영역에서 스크롤바 숨김 + 스냅 정상 동작 확인

**10-5. PC 대응 (선택, 이후 확장)**
- 모바일 완성 후 필요시 진행
- `max-width: 430px` + `margin: 0 auto`로 중앙 정렬 프레임
- 데스크톱 배경: `--color-cream` 또는 연한 패턴
- 폰 프레임 그림자 등 시각적 래핑 (선택)

**10-6. 접근성 기본**
- 버튼에 `aria-label` 추가
- 포커스 스타일 유지 (outline 제거 금지)
- 이미지 placeholder에 `alt` 텍스트

**10-6. 경로 확인**
- 모든 CSS, JS, JSON, 이미지 경로가 상대 경로인지 검증
- `/`로 시작하는 절대 경로 없는지 확인

**10-7. GitHub Pages 배포**
- GitHub repo 생성 + 코드 push
- Settings → Pages → Source: `main` / `/ (root)` 선택
- 배포 URL 확인: `https://<username>.github.io/<repo>`
- 모바일 브라우저에서 실제 접속 테스트

**10-8. README.md 작성**
- 프로젝트 소개, 스크린샷, 기술 스택, 실행 방법
- 라이브 데모 링크

### 완료 조건
- [ ] 모든 탭, 서브 페이지, 인터랙션이 에러 없이 동작한다
- [ ] 전환 애니메이션이 자연스럽게 적용된다
- [ ] GitHub Pages URL에서 앱이 정상 로드된다
- [ ] **모바일 브라우저(Chrome, Safari)에서 모든 화면이 깔끔하게 보인다** (1차 완료 기준)
- [ ] 콘솔에 에러가 없다
- [ ] (선택) PC에서 접속 시 모바일 프레임이 중앙 정렬된다

STEP 1 (앱 셸)
